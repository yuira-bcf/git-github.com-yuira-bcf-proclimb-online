Section 3-1

演算子と優先順位

1.1 算術演算子と単項演算子

演算子 説明 例
+加算5 + 3 → 8
-減算5 - 3 → 2
*乗算5 * 3 → 15
/除算5 / 3 → 1（整数除算）
%剰余5 % 3 → 2
++インクリメントi++, ++i
--デクリメントi--, --i
前置と後置の違い Java 
int a = 5;
int b = a++;  // b = 5, a = 6（代入後にインクリメント）

int c = 5;
int d = ++c;  // d = 6, c = 6（インクリメント後に代入）

1.2 代入演算子と複合代入演算子

複合代入演算子 Java 
int x = 10;
x += 5;   // x = x + 5  → 15
x -= 3;   // x = x - 3  → 12
x *= 2;   // x = x * 2  → 24
x /= 4;   // x = x / 4  → 6
x %= 4;   // x = x % 4  → 2

1.3 関係演算子

演算子 説明 例
==等しい5 == 5 → true
!=等しくない5 != 3 → true
<より小さい3 < 5 → true
>より大きい5 > 3 → true
<=以下5 <= 5 → true
>=以上5 >= 3 → true

1.4 論理演算子

演算子 説明 例
&&論理AND（短絡評価）true && false → false
||論理OR（短絡評価）true || false → true
!論理NOT!true → false
&論理AND（完全評価）true & false → false
|論理OR（完全評価）true | false → true
💡 短絡評価とは

&&は左辺がfalseなら右辺を評価しません。||は左辺がtrueなら右辺を評価しません。これにより効率的な評価とNullPointerExceptionの回避ができます。

1.5 条件演算子（三項演算子）

条件演算子 Java 
// 構文: 条件 ? 真の場合の値 : 偽の場合の値
int a = 10, b = 20;
int max = (a > b) ? a : b;  // max = 20

String status = (score >= 60) ? "合格" : "不合格";
Section 3-2

データの比較

2.1 基本データ型と参照型の比較

==演算子の動作 Java 
// 基本データ型：値を比較
int a = 10;
int b = 10;
System.out.println(a == b);  // true

// 参照型：参照（アドレス）を比較
String s1 = new String("Hello");
String s2 = new String("Hello");
System.out.println(s1 == s2);  // false（異なるオブジェクト）
System.out.println(s1.equals(s2));  // true（内容が同じ）

2.2 Stringクラスの特徴と文字列の比較

文字列プール Java 
// 文字列リテラルは文字列プールで共有される
String s1 = "Hello";
String s2 = "Hello";
System.out.println(s1 == s2);  // true（同じオブジェクト）

// newで作成すると新しいオブジェクト
String s3 = new String("Hello");
System.out.println(s1 == s3);  // false

// 文字列比較には必ずequals()を使用
System.out.println(s1.equals(s3));  // true
⚠️ 重要

文字列の比較には常にequals()メソッドを使用してください。==は参照の比較であり、意図しない結果になることがあります。

2.4 nullの比較

nullの扱い Java 
String str = null;

// nullとの比較は==を使用
if (str == null) {
    System.out.println("strはnullです");
}

// nullに対してメソッドを呼ぶとNullPointerException
// str.length();  // 実行時エラー

// 安全な比較方法
if (str != null && str.length() > 0) {
    // 短絡評価により、strがnullの場合length()は呼ばれない
}

// Objects.equals()を使う方法（Java 7以降）
import java.util.Objects;
Objects.equals(str, "Hello");  // strがnullでもOK
Section 3-3

基本データ型の型変換

3.1 暗黙の型変換（拡大変換）

小さい型から大きい型への変換は自動的に行われます。

暗黙の型変換 Java 
byte b = 10;
int i = b;      // byte → int（自動変換）

int n = 100;
long l = n;     // int → long（自動変換）

float f = 3.14f;
double d = f;   // float → double（自動変換）

変換の順序: byte → short → int → long → float → double

3.2 キャストによる型変換（縮小変換）

大きい型から小さい型への変換は明示的なキャストが必要です。

キャスト Java 
int i = 100;
byte b = (byte) i;  // int → byte（明示的キャスト）

double d = 3.14;
int n = (int) d;    // double → int（小数部切り捨て）→ 3

long l = 1000L;
int x = (int) l;    // long → int（明示的キャスト）
⚠️ データ損失に注意

縮小変換では値が失われる可能性があります。例：(byte) 128 は -128 になります（オーバーフロー）。

3.4 型変換の注意点

演算時の型変換 Java 
// 整数同士の除算は整数結果
int a = 5, b = 2;
System.out.println(a / b);  // 2（小数部切り捨て）

// 一方をdoubleにすると小数結果
System.out.println((double) a / b);  // 2.5
System.out.println(a / (double) b);  // 2.5

// byte/short/charの演算結果はintになる
byte x = 10, y = 20;
// byte z = x + y;  // コンパイルエラー
int z = x + y;      // OK
byte w = (byte)(x + y);  // キャストが必要

4.2 ボクシングとアンボクシング

オートボクシング/アンボクシング Java 
// オートボクシング（基本型 → ラッパー）
Integer num = 100;  // int → Integer

// オートアンボクシング（ラッパー → 基本型）

int i2 = i1.intValue();            // アンボクシング

ラッパークラスはnullを持てます。nullをアンボクシングするとNullPointerExceptionが発生します。

// 数値 → 文字列
String s1 = String.valueOf(123);
String s2 = Integer.toString(123);
String s3 = "" + 123;  // 文字列連結を利用

条件分岐とifによる分岐

6.1 if文

基本的なif文 Java 
int score = 85;

if (score >= 60) {
    System.out.println("合格です");
}

6.2 if-else文

if-else文 Java 
int score = 45;

if (score >= 60) {
    System.out.println("合格です");
} else {
    System.out.println("不合格です");
}

6.3 if-else if文

複数条件の分岐 Java 
int score = 75;
String grade;

if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
} else if (score >= 70) {
    grade = "C";
} else if (score >= 60) {
    grade = "D";
} else {
    grade = "F";
}
System.out.println("評価: " + grade);  // 評価: C

6.4 if文のネスト

入れ子のif文 Java 
int age = 25;
boolean hasLicense = true;

if (age >= 18) {
    if (hasLicense) {
        System.out.println("運転可能です");
    } else {
        System.out.println("免許を取得してください");
    }
} else {
    System.out.println("18歳以上になってから");
}
Section 3-7

switchによる分岐

7.1 switch文

従来のswitch文 Java 
int day = 3;
String dayName;

switch (day) {
    case 1:
        dayName = "月曜日";
        break;
    case 2:
        dayName = "火曜日";
        break;
    case 3:
        dayName = "水曜日";
        break;
    case 4:
        dayName = "木曜日";
        break;
    case 5:
        dayName = "金曜日";
        break;
    default:
        dayName = "週末";
        break;
}
System.out.println(dayName);  // 水曜日
⚠️ break忘れに注意

breakを忘れると次のcaseも実行されます（フォールスルー）。意図的でない限りbreakを忘れないようにしましょう。

7.2 ->を使用したswitchのラベル（Java 14以降）

アロー構文のswitch Java 
int day = 3;
String dayName;

switch (day) {
    case 1 -> dayName = "月曜日";
    case 2 -> dayName = "火曜日";
    case 3 -> dayName = "水曜日";
    case 4 -> dayName = "木曜日";
    case 5 -> dayName = "金曜日";
    case 6, 7 -> dayName = "週末";  // 複数のcase
    default -> dayName = "不正な値";
}

アロー構文ではbreakは不要で、フォールスルーは発生しません。

7.3 switch式（Java 14以降）

switch式 Java 
int day = 3;

// switch式は値を返す
String dayName = switch (day) {
    case 1 -> "月曜日";
    case 2 -> "火曜日";
    case 3 -> "水曜日";
    case 4 -> "木曜日";
    case 5 -> "金曜日";
    case 6, 7 -> "週末";
    default -> "不正な値";
};

// 複数行の場合はyieldで値を返す
String description = switch (day) {
    case 1, 2, 3, 4, 5 -> {
        System.out.println("平日です");
        yield "仕事の日";
    }
    case 6, 7 -> {
        System.out.println("休日です");
        yield "休みの日";
    }
    default -> "不正";
};
Exercise

3章 練習問題

Q1

以下のコードの出力は何ですか？

int a = 5;
int b = a++;
System.out.println(a + " " + b);
  1. 5 5
  2. 6 5
  3. 5 6
  4. 6 6
Q2

文字列の比較として正しいものはどれですか？

  1. str1 == str2
  2. str1.equals(str2)
  3. str1.compareTo(str2) == 0
  4. BとCの両方
Q3

以下のコードの出力は何ですか？

int x = 5 / 2;
double y = 5 / 2;
System.out.println(x + " " + y);
  1. 2 2.5
  2. 2.5 2.5
  3. 2 2.0
  4. 2.5 2.0
Answer

3章 解説・解答

Q1の解答: B

後置インクリメントa++は、現在の値を返してからインクリメントします。したがって、bには5が代入され、その後aが6になります。

Q2の解答: D

equals()は内容の等価性を比較します。compareTo()が0を返す場合も内容が等しいことを意味します。==は参照の比較なので文字列の比較には適していません。

Q3の解答: C

整数同士の除算5 / 2は整数結果の2になります。これをdoubleに代入しても2.0です。2.5にするには5.0 / 2や(double)5 / 2とする必要があります。
