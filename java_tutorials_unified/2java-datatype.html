Section 2-1

リテラル

リテラルとは、ソースコード中に直接記述される値のことです。数値、文字、文字列などがあります。

1.1 数値リテラル

種類 例 説明
整数（10進数） 123 通常の整数表記
整数（2進数） 0b1010 0bまたは0Bで始まる
整数（8進数） 0755 0で始まる
整数（16進数） 0xFF 0xまたは0Xで始まる
long型 123L 末尾にLまたはlを付ける
浮動小数点（double） 3.14 デフォルトはdouble型
浮動小数点（float） 3.14f 末尾にFまたはfを付ける
💡 アンダースコア区切り

Java 7以降、数値リテラルにアンダースコアを入れて読みやすくできます：1_000_000

1.2 文字と文字列リテラル

文字と文字列 Java 
// 文字リテラル（シングルクォート）
char c = 'A';

// 文字列リテラル（ダブルクォート）
String s = "Hello";

// エスケープシーケンス
String newline = "Line1\nLine2";  // 改行
String tab = "Column1\tColumn2";  // タブ
String quote = "He said \"Hello\"";  // ダブルクォート
エスケープシーケンス 意味
\n改行
\tタブ
\\バックスラッシュ
\"ダブルクォート
\'シングルクォート

1.3 テキストブロック

Java 15で正式導入されたテキストブロックを使うと、複数行の文字列を簡単に記述できます。

テキストブロック（Java 15以降） Java 
String html = """
    <html>
        <body>
            <p>Hello, World!</p>
        </body>
    </html>
    """;

1.4 その他のリテラル

真偽値とnull Java 
// 真偽値リテラル
boolean flag1 = true;
boolean flag2 = false;

// nullリテラル（参照型の「何も参照していない」状態）
String str = null;
Section 2-2

変数や定数の宣言と初期化

2.1 変数と命名規則

変数名のルール：

  • 英字、数字、アンダースコア、ドル記号が使用可能
  • 数字で始めることはできない
  • 予約語は使用できない
  • 大文字と小文字は区別される
命名の例 Java 
// 有効な変数名
int age;
String firstName;
double _value;
int $amount;

// 無効な変数名
// int 1stPlace;  // 数字で始まる
// int class;     // 予約語

2.2 データ型

Javaのデータ型は基本データ型（プリミティブ型）と参照型に分類されます。

基本データ型（8種類）

分類 型 サイズ 範囲/説明
整数 byte 1バイト -128 ～ 127
short 2バイト -32,768 ～ 32,767
int 4バイト 約-21億 ～ 約21億
long 8バイト 約-922京 ～ 約922京
浮動小数点 float 4バイト 単精度浮動小数点
double 8バイト 倍精度浮動小数点
文字 char 2バイト Unicode文字（0～65,535）
真偽値 boolean - true または false

2.3 変数宣言と値の代入

変数の宣言と初期化 Java 
// 宣言と初期化を分ける
int number;
number = 10;

// 宣言と初期化を同時に
int age = 25;

// 複数の変数を同時に宣言
int a, b, c;
int x = 1, y = 2, z = 3;
⚠️ ローカル変数の初期化

ローカル変数は使用前に必ず初期化が必要です。初期化されていない変数を使用するとコンパイルエラーになります。

2.4 定数

finalキーワードを使って定数を宣言します。定数は一度値を設定すると変更できません。

定数の宣言 Java 
// 定数（慣習的に大文字とアンダースコアで命名）
final double PI = 3.14159;
final int MAX_SIZE = 100;

// PI = 3.14;  // コンパイルエラー：再代入不可

2.5 文字列の型

Stringは参照型ですが、特別な扱いを受けます。

String型 Java 
// 文字列リテラルで初期化
String s1 = "Hello";

// newキーワードで初期化（非推奨）
String s2 = new String("Hello");

// 文字列の連結
String greeting = "Hello" + " " + "World";
Section 2-3

ローカル変数の型推論

Java 10以降、varキーワードを使ってローカル変数の型を推論できます。

varによる型推論（Java 10以降） Java 
// varを使った宣言
var number = 10;           // int型と推論
var name = "Alice";        // String型と推論
var list = new ArrayList<String>();  // ArrayList<String>型と推論

// varが使えない場合
// var x;                  // 初期化が必要
// var y = null;           // 型が推論できない
// var z = {1, 2, 3};      // 配列初期化子は不可
💡 varの使用場面

varはローカル変数にのみ使用可能です。フィールド、メソッドの引数、戻り値の型には使用できません。

Section 2-4

配列

4.1 1次元配列

配列の宣言と初期化 Java 
// 配列の宣言
int[] numbers;
String[] names;

// 配列の生成
numbers = new int[5];  // 要素数5の配列

// 宣言と生成を同時に
int[] scores = new int[3];

// 初期化子を使った初期化
int[] values = {1, 2, 3, 4, 5};
int[] data = new int[]{10, 20, 30};

// 配列の要素にアクセス
values[0] = 100;  // 最初の要素に代入
int first = values[0];  // 最初の要素を取得

// 配列の長さ
int length = values.length;  // 5
⚠️ 配列の初期値

配列をnewで生成すると、要素は自動的にデフォルト値で初期化されます（int:0、boolean:false、参照型:null）。

4.2 多次元配列

2次元配列 Java 
// 2次元配列の宣言と生成
int[][] matrix = new int[3][4];  // 3行4列

// 初期化子を使った初期化
int[][] grid = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

// 要素へのアクセス
matrix[0][0] = 10;  // 0行0列目
int value = grid[1][2];  // 1行2列目（値は6）

// ジャグ配列（各行の長さが異なる配列）
int[][] jagged = new int[3][];
jagged[0] = new int[2];
jagged[1] = new int[4];
jagged[2] = new int[3];
Section 2-5

コマンドライン引数の利用

mainメソッドの引数String[] argsでコマンドライン引数を受け取れます。

コマンドライン引数 Java 
public class CommandLineDemo {
    public static void main(String[] args) {
        // 引数の数を確認
        System.out.println("引数の数: " + args.length);

        // 各引数を表示
        for (int i = 0; i < args.length; i++) {
            System.out.println("args[" + i + "] = " + args[i]);
        }
    }
}
実行例 Terminal 
$ java CommandLineDemo Hello World 123
引数の数: 3
args[0] = Hello
args[1] = World
args[2] = 123
Section 2-6

文字列の操作

6.1 Stringクラスの特徴

Stringクラスの重要な特徴：

  • イミュータブル（不変）：一度作成された文字列は変更できない
  • 文字列プール：同じ文字列リテラルは共有される
Stringの不変性 Java 
String s = "Hello";
s.toUpperCase();  // 元の文字列は変更されない
System.out.println(s);  // "Hello"

s = s.toUpperCase();  // 新しい文字列を代入
System.out.println(s);  // "HELLO"

6.2 Stringクラスのメソッド

メソッド 説明 例
length() 文字列の長さ "Hello".length() → 5
charAt(int) 指定位置の文字 "Hello".charAt(0) → 'H'
substring(int) 部分文字列 "Hello".substring(2) → "llo"
indexOf(String) 文字列の位置 "Hello".indexOf("l") → 2
toUpperCase() 大文字変換 "Hello".toUpperCase() → "HELLO"
toLowerCase() 小文字変換 "Hello".toLowerCase() → "hello"
trim() 前後の空白除去 " Hi ".trim() → "Hi"
replace(old, new) 置換 "Hello".replace('l', 'L') → "HeLLo"
equals(Object) 等価比較 "Hello".equals("Hello") → true
contains(String) 含むか判定 "Hello".contains("ell") → true
startsWith(String) 先頭判定 "Hello".startsWith("He") → true
split(String) 分割 "a,b,c".split(",") → ["a","b","c"]

6.3 テキストブロックの操作

テキストブロックのメソッド Java 
String text = """
    Hello
    World
    """;

// インデントの調整
String stripped = text.stripIndent();

// フォーマット
String formatted = """
    Name: %s
    Age: %d
    """.formatted("Alice", 25);

6.4 StringBuilderクラス

頻繁に文字列を変更する場合は、StringBuilderを使用すると効率的です。

StringBuilder Java 
StringBuilder sb = new StringBuilder();

// 文字列の追加
sb.append("Hello");
sb.append(" ");
sb.append("World");

// メソッドチェーン
sb.append("!").append(" ").append("Java");

// 挿入
sb.insert(0, "[");  // 先頭に挿入

// 削除
sb.delete(0, 1);    // 範囲削除

// 置換
sb.replace(0, 5, "Hi");

// 反転
sb.reverse();

// Stringに変換
String result = sb.toString();
💡 StringBufferとの違い

StringBufferはスレッドセーフですが、StringBuilderより遅いです。シングルスレッド環境ではStringBuilderを推奨します。

Exercise

2章 練習問題

Q1

次のうち、有効な変数宣言はどれですか？（複数選択）

  1. int 2ndValue = 10;
  2. String _name = "test";
  3. double $price = 100.0;
  4. boolean class = true;
Q2

以下のコードの出力は何ですか？

String s = "Hello";
s.toUpperCase();
System.out.println(s);
  1. HELLO
  2. Hello
  3. hello
  4. コンパイルエラー
Q3

int[] arr = new int[3];で生成された配列の各要素の初期値は何ですか？

  1. null
  2. 0
  3. 未定義（ランダム）
  4. コンパイルエラー
Answer

2章 解説・解答

Q1の解答: B, C

A: 変数名は数字で始められません。D: classは予約語です。B: アンダースコアで始まる名前は有効です。C: ドル記号で始まる名前も有効です。

Q2の解答: B

Stringはイミュータブル（不変）なので、toUpperCase()は新しい文字列を返しますが、元の変数sは変更されません。結果を反映するにはs = s.toUpperCase();とする必要があります。

Q3の解答: B

int型配列をnewで生成すると、各要素はデフォルト値の0で初期化されます。参照型の場合はnull、booleanの場合はfalseになります。
