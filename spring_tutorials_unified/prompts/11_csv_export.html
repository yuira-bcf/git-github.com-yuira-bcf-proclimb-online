Section 11-2

CSVファイルの生成

TodoController.java 
import jakarta.servlet.http.HttpServletResponse;
import java.io.PrintWriter;
import java.net.URLEncoder;
import java.nio.charset.StandardCharsets;

/**
 * CSVエクスポート
 */
@GetMapping("/export")
public void exportCsv(HttpServletResponse response) throws Exception {
    List<Todo> todoList = todoService.findAll();

    // ファイル名（日本語対応）
    String filename = URLEncoder.encode("todo_list.csv", StandardCharsets.UTF_8);

    // レスポンスヘッダー設定
    response.setContentType("text/csv; charset=UTF-8");
    response.setHeader("Content-Disposition", "attachment; filename=\"" + filename + "\"");

    // BOM付きUTF-8（Excelで文字化けしないため）
    PrintWriter writer = response.getWriter();
    writer.print('\ufeff');  // BOM

    // ヘッダー行
    writer.println("No,タイトル,登録者,詳細,作成日");

    // データ行
    for (Todo todo : todoList) {
        writer.println(String.format("%d,\"%s\",\"%s\",\"%s\",\"%s\"",
            todo.getId(),
            escapeCsv(todo.getTitle()),
            escapeCsv(todo.getAuthor()),
            escapeCsv(todo.getDetail()),
            todo.getCreatedAt() != null ?
                todo.getCreatedAt().format(DateTimeFormatter.ofPattern("yyyy/MM/dd")) : ""
        ));
    }
}

// CSV用エスケープ
private String escapeCsv(String value) {
    if (value == null) return "";
    return value.replace("\"", "\"\"");
}

Section 11-3

文字化け対策

ExcelでCSVを開くときの問題

対策説明
BOM付きUTF-8\ufeffを先頭に付与
ダブルクォート囲みカンマを含む値を正しく処理
ダブルクォートエスケープ"を""に置換
⚠️BOMについて

BOM（Byte Order Mark）を付けると、ExcelがUTF-8として認識します。
テキストエディタで開くと先頭に不可視文字が入りますが、正常な動作です。

🤖 AI

AIペアプログラミング

以下のプロンプトをAIアシスタント（Claude、ChatGPT等）に入力することで、この機能を実装できます。

📝 基本プロンプト 
Spring Boot 3.xのToDoアプリにCSVエクスポート機能を追加してください。

【要件】
- 「CSVダウンロード」ボタンを一覧画面に追加
- クリックでToDoデータをCSV形式でダウンロード
- CSVの項目：ID, タイトル, 登録者, ステータス, 作成日
- 文字コードはUTF-8（BOM付き）でExcel対応
- ファイル名はtodo_yyyyMMdd.csv形式

【技術スタック】
- Spring Boot 3.x (ResponseEntity, HttpHeaders)
- Java CSV出力（OpenCSV等は使用しない）
🎯 ステップ別プロンプト 
# Step 1: CSVエクスポート用エンドポイントの作成
GET /todo/exportエンドポイントを作成し、ResponseEntity<byte[]>でCSVデータを返すようにしてください。

# Step 2: CSVデータの生成
ToDoリストをCSV形式の文字列に変換するメソッドを作成してください。カンマやダブルクォートのエスケープ処理も行ってください。

# Step 3: HTTPヘッダーの設定
Content-Typeをtext/csv、Content-Dispositionでファイル名を設定し、ダウンロードさせてください。

# Step 4: Excel対応（BOM付きUTF-8）
CSVデータの先頭にBOM（0xEF, 0xBB, 0xBF）を付与して、Excelで文字化けしないようにしてください。
💡 プロンプトのコツ

「現在の検索条件を維持したままCSVエクスポートできるようにしてください」のように、
既存機能との連携を指定すると、実用的な実装が得られます。
