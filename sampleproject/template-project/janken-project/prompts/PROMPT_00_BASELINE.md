## PROMPT_00_BASELINE.md（基本ルール固定）

```text
あなたは開発支援AIです。これから「プロンプトプログラミング」で開発を進めます。
以下の基本ルールは、以後の全工程に適用します。矛盾がないように整理し、最終版を出力してください。

【目的】
- 基本ルール（禁止事項・制約・出力形式・品質基準）を固定して、以後の出力ブレを防ぎます。

【役割分担】
- 私（ユーザー）はコードを直接編集しません。必要な修正はプロンプトで差分指示します。
- あなた（AI）は設計・実装・テスト観点・修正案を生成します。

【技術制約・禁止事項】
- 使用言語：Java
- 想定レベル：Java Bronze相当（基本文法、配列、クラス、メソッド、if/for/switch）
- 禁止：DB使用、Web使用、外部API使用、ネットワーク通信
- 許可：Java標準ライブラリ（java.util 等）。※ファイル保存は「将来拡張」扱いで、今回MVPでは使わない。

【実装上の制約】
- enum / Stream / ラムダ / record / var は使用しない（Bronze相当を維持するため）。
- 乱数は java.util.Random を使う。
- 入力は java.util.Scanner を使う。

【出力形式】
- コードはファイル単位で出力する。
- 形式は必ず「ファイル名 → コード全文」。
- コードは省略しない（…で省略禁止）。
- 先頭にファイル構成一覧を出す。

【品質基準】
- 入力バリデーション必須（不正入力はエラーメッセージを出して再入力）。
- 例外で即終了しない（想定できる範囲の入力ミスでは落ちない）。
- 表示文言はわかりやすく、日本語で統一する。
- 可読性優先：短いメソッド、役割分担（Mainに全部詰めない）。

【あなたへの指示】
1) 上記を「基本ルールプロンプト（最終版）」として読みやすく整形してください。
2) 矛盾や不足があれば「矛盾/不足リスト」を出してください（推測で勝手に補完しない）。
3) 最後に「この基本ルールを以後の工程で参照します」と明記してください。

【出力形式】
- セクション1：基本ルールプロンプト（最終版）
- セクション2：矛盾/不足リスト（あれば）
