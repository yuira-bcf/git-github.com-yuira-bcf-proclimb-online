メソッドとは（概要・定義・呼び出し・mainメソッドから呼び出し）

1. 概要（メソッドとは何か）

Java におけるメソッドとは、処理のまとまりのことです。 よく使う処理に名前を付けて、必要なときに呼び出せるようにします。

メソッド = 「処理をまとめて名前を付けたもの」

メソッドを使うと、次のようなメリットがあります。

  • 同じ処理を何度も書かずに済みます（再利用できます）
  • 処理が短くまとまり、読みやすくなります
  • 役割ごとに分けられるため、修正しやすくなります

2. 定義（メソッドの書き方）

メソッドはクラスの中に定義します。基本的な形は次の通りです。 

戻り値の型 メソッド名(引数) {
    処理
}

例として、「あいさつを表示するメソッド」を定義してみます。

public class Main {

    // メソッドの定義（引数なし、戻り値なし）
    static void hello() {
        System.out.println("こんにちは");
    }

}

ポイント：

  • hello がメソッド名です（好きな名前を付けられます）
  • void は「戻り値がない」という意味です
  • () の中が空なら「引数なし」です

3. 呼び出し（メソッドの使い方）

メソッドは、定義しただけでは動きません。呼び出したときに実行されます。 

メソッド名();

先ほど定義した hello を呼び出すと、次のようになります。

hello();

イメージ：
メソッドは「ボタン」のようなものだと考えると分かりやすいです。
定義は「ボタンを作る」、呼び出しは「ボタンを押す」です。

4. mainメソッドから呼び出す

Java のプログラムは、通常 main メソッドから処理が始まります。 そのため、初学者の方はまず 「main から自分のメソッドを呼び出す」ことを練習すると理解が進みます。

次の例では、main メソッドの中で hello を呼び出しています。

public class Main {

    static void hello() {
        System.out.println("こんにちは");
    }

    public static void main(String[] args) {
        // main からメソッドを呼び出す
        hello();
    }
}

実行結果の例：

こんにちは

補足（初学者向け）：
この例で hello の前に Main. を付けていないのは、
main と hello が同じクラスの中にあり、どちらも static だからです。
（この段階では「そういうルール」くらいで大丈夫です）

まとめ

メソッドは「処理をまとめて名前を付けたもの」です。定義したメソッドは、呼び出したときに実行されます。初学者はまず、mainメソッドから自分のメソッドを呼び出す形を練習するのが効果的です。

引数の利用

1. 引数とは

引数（ひきすう）とは、メソッドに値を渡すための仕組みです。

メソッドは「処理のまとまり」ですが、引数を使うことで 外から値を受け取り、内容を変えて実行できます。

引数とは「メソッドに渡すデータ」

2. 1つの引数を渡す

2-1. 定義

1つの引数を受け取るメソッドは、次のように定義します。 

static void greet(String name) {
    System.out.println(name + "さん、こんにちは");
}

ポイント：
String name が引数です。
この name は、メソッドの中だけで使える変数です。

2-2. 呼び出し

引数を持つメソッドは、呼び出すときに値を渡します。 

greet("田中");

実行結果：

田中さん、こんにちは

3. 複数の引数を渡す

3-1. 定義

複数の引数を使う場合は、カンマ（,）で区切って定義します。 

static void add(int a, int b) {
    System.out.println(a + b);
}

3-2. 呼び出し

呼び出すときも、同じ順番で値を渡します。 

add(10, 20);

実行結果：

30

注意：
引数は「順番」が重要です。
型と順番が一致していないと、コンパイルエラーになります。

4. 仮引数と実引数

引数には、呼び方の違いがあります。

  • 仮引数：メソッド定義側の引数
  • 実引数：メソッド呼び出し時に渡す値
// 仮引数
static void greet(String name) {
    System.out.println(name);
}

// 実引数
greet("佐藤");

整理すると：
・String name → 仮引数
・"佐藤" → 実引数

5. 変数のスコープとローカル変数

スコープとは、「変数が使える範囲」のことです。

5-1. ローカル変数

メソッドの中で定義された変数は、 そのメソッドの中だけで使えます。 

static void sample() {
    int x = 10;   // ローカル変数
    System.out.println(x);
}

上記の x は、sample メソッドの外では使えません。

重要：
引数もローカル変数の一種です。
メソッドの外から直接使うことはできません。

まとめ

引数を使うことで、メソッドは外から値を受け取り、柔軟に処理できるようになります。 仮引数と実引数の違い、そして変数のスコープを理解することが、 正しいプログラムを書くための重要なポイントです。

戻り値の利用

1. 戻り値とは

戻り値とは、メソッドが処理を行った結果として、 呼び出し元に返す値のことです。

戻り値 = メソッドの実行結果

これまで学習した void のメソッドは、 処理は行いますが値を返しません。 戻り値を使うことで、計算結果などを後続の処理に利用できます。

定義

戻り値を持つメソッドは、 void の代わりに 返したい値の型を書いて定義します。 

static int add(int a, int b) {
    return a + b;
}

ポイント：
・int が戻り値の型です。
・return は値を返して、メソッドを終了します。

2. 戻り値を受け取る

戻り値を持つメソッドは、 変数に代入して受け取ることができます。

定義（受け取り方）

int result = add(10, 20);

この処理では、次の順序で動いています。

  • add(10, 20) が実行される
  • 計算結果 30 が返される
  • result に 30 が代入される

受け取った値は、普通の変数として利用できます。

System.out.println(result);

3. 戻り値をそのまま使う

戻り値は、必ずしも変数に入れる必要はありません。 そのまま別の処理に使うこともできます。 

System.out.println(add(10, 20));

この場合、 add(10, 20) の戻り値が、 直接 println に渡されています。

考え方：
「戻り値があるメソッドは、値として使える」

4. return 文の注意点

① return を実行するとメソッドは終了する

static int sample() {
    return 10;
    // この下の処理は実行されません
}

注意：
return が実行された時点で、 メソッドの処理はそこで終わります。

② 戻り値の型と一致している必要がある

// 正しい例
static int sample() {
    return 5;
}

// 間違いの例（型が違う）
static int sample() {
    return "5";
}

③ 戻り値があるメソッドでは return が必須

戻り値の型を指定した場合、 必ず return 文で値を返す必要があります。 

// コンパイルエラーになる例
static int sample() {
    int x = 10;
}

まとめ

戻り値とは、メソッドが処理結果として返す値です。 戻り値は変数に代入して受け取ることも、そのまま使うこともできます。 return 文は値を返すと同時に、メソッドを終了させる重要な役割を持っています。

オーバーロードの利用

1. オーバーロードとは

オーバーロード（Overload）とは、 同じ名前のメソッドを、引数の違いによって複数定義することです。

同じ処理の名前を使い、
引数の数や型が違うメソッドを用意する仕組み

処理内容が似ている場合でも、 メソッド名を増やさず整理できるため、 プログラムが読みやすくなります。

2. 類似する複数メソッドを定義する

例えば「値を表示する処理」を考えます。 表示したい値が int の場合と String の場合で処理内容はほぼ同じです。 

// オーバーロードを使った例
static void print(int x) {
    System.out.println(x);
}

static void print(String s) {
    System.out.println(s);
}

呼び出し側では、引数を渡すだけで、 Java が自動的に適切なメソッドを選択します。 

print(100);
print("Java");

ポイント：
メソッド名は同じでも、 引数の型が違うため区別できます。

3. メソッドのシグネチャ

Java が「どのメソッドを呼ぶか」を判断する基準を メソッドのシグネチャと呼びます。

メソッドのシグネチャ ＝ メソッド名 ＋ 引数の型と順番

次の 2 つのメソッドは、 引数の型が異なるため、同時に定義できます。 

static void calc(int a, int b) {
}

static void calc(double a, double b) {
}

一方、次のような定義はできません。 

// コンパイルエラーになる例
static int sample(int x) {
    return x;
}

static void sample(int x) {
}

注意：
戻り値の型はシグネチャに含まれません。
引数が同じで戻り値だけが違うメソッドは定義できません。

4. サンプルプログラム（全体例）

次は、オーバーロードを使った 完全なサンプルプログラムです。 

public class Main {

    // int を受け取って表示するメソッド
    static void show(int x) {
        System.out.println("int型: " + x);
    }

    // String を受け取って表示するメソッド
    static void show(String s) {
        System.out.println("String型: " + s);
    }

    // int を2つ受け取って表示するメソッド
    static void show(int a, int b) {
        System.out.println("int型2つ: " + a + ", " + b);
    }

    public static void main(String[] args) {
        show(10);
        show("Hello");
        show(3, 5);
    }
}

実行結果の例：

int型: 10
String型: Hello
int型2つ: 3, 5

確認ポイント：
・すべて同じ show という名前のメソッドです。
・引数の数や型に応じて、呼び出されるメソッドが変わります。

まとめ

オーバーロードとは、引数の違いによって 同じ名前のメソッドを複数定義する仕組みです。 Java はメソッドのシグネチャ（メソッド名と引数）を使って 呼び出すメソッドを判断します。 サンプルプログラムのように、処理が似ている場合に活用すると、 コードを分かりやすく整理できます。

引数や戻り値に配列を用いる

このセクションで押さえる結論

Java は常に「値渡し」です。
ただし配列の場合、コピーされるのは「配列を指す情報（参照）」なので、
同じ配列の中身を変更すると return がなくても呼び出し元に反映されます。

1. 引数に配列を用いる

1-1. 概要

メソッドに配列を渡すと、メソッドの中で 複数の値をまとめて扱うことができます。 たとえば、合計・平均・最大値の計算などを配列1つで渡せます。

1-2. 定義方法

配列を引数にする場合は、引数の型を配列型（型[]）にします。 

// int 配列を受け取る
static void printAll(int[] nums) {
    // 処理
}

1-3. サンプルプログラム

配列の中身をすべて表示する例です。 

public class Main {

    // 配列を受け取り、要素をすべて表示する
    static void printAll(int[] nums) {
        for (int i = 0; i < nums.length; i++) {
            System.out.println(nums[i]);
        }
    }

    public static void main(String[] args) {
        int[] data = {10, 20, 30};
        printAll(data);
    }
}

実行結果：

10
20
30

2. 値渡しと参照渡し

2-1. 概要

まず重要な事実として、Java ではメソッド呼び出しの際に 実引数の値がコピーされ、仮引数に渡されます。

仮引数と実引数の関係：

  • 実引数：呼び出し側が渡す値（例：data）
  • 仮引数：メソッド側が受け取る変数（例：array）
  • 渡されるときに値のコピーが起きます

では、配列の場合は何がコピーされるのでしょうか。 配列を変数に入れると、変数には「配列の中身」ではなく 配列を指す情報（参照）が入ります。 その参照がコピーされるため、結果として「参照渡しのように見える」動作になります。

2-2. 定義方法

配列を引数に取るメソッドは、次のように定義します。 

static void incArray(int[] array) {
    // 配列の要素を変更する処理
}

2-3. サンプルプログラム（return なしでも呼び出し元が変わる）

次の例では return を書いていませんが、 data[0] が 100 に変わります。 

public class Main {

    // 配列の先頭要素を 100 に変更する
    static void incArray(int[] array) {
        array[0] = 100;
    }

    public static void main(String[] args) {
        int[] data = {1, 2, 3};

        // 実引数 data の「参照」がコピーされ、仮引数 array に渡される
        incArray(data);

        // 同じ配列の中身を書き換えたため、呼び出し元でも変化が見える
        System.out.println(data[0]);
    }
}

実行結果：

100
配列で return が不要なのは、
「同じ配列の要素（中身）を直接変更している」ためです。

注意：

次のように「仮引数の参照を別の配列に付け替える」操作は、 呼び出し元の data を変えません。 

static void changeRef(int[] array) {
    array = new int[]{9, 9, 9}; // 参照の付け替え（呼び出し元には影響しない）
}

3. 戻り値に配列を用いる

3-1. 概要

配列を戻り値にすると、 複数の値をまとめて返すことができます。 たとえば「1〜n の一覧」「計算結果の一覧」などを返せます。

3-2. 定義方法

戻り値の型を配列型（型[]）にします。 

// int 配列を返す
static int[] makeArray(int size) {
    // 配列を作成して return する
}

戻り値として返るのは、配列の中身そのものではなく、 配列を指す情報（参照）です。 呼び出し元は、その参照を受け取って配列を利用します。

3-3. サンプルプログラム

0 から size - 1 までを入れた配列を作って返す例です。 

public class Main {

    // size 個の配列を作り、0,1,2... を入れて返す
    static int[] makeArray(int size) {
        int[] newArray = new int[size];
        for (int i = 0; i < newArray.length; i++) {
            newArray[i] = i;
        }
        return newArray;
    }

    public static void main(String[] args) {
        int[] data = makeArray(3);

        for (int x : data) {
            System.out.println(x);
        }
    }
}

実行結果：

0
1
2

まとめ

・配列を引数にすると、複数の値をまとめて渡せます。
・Java は常に値渡しであり、実引数の値がコピーされて仮引数に渡されます。
・配列の場合、コピーされるのは「配列を指す情報（参照）」です。
・そのため、メソッド内で配列の要素を変更すると return がなくても呼び出し元に反映されます。
・配列を戻り値にすると、複数の結果をまとめて返すことができます（返るのは参照情報です）。

コマンドライン引数とは

1. コマンドライン引数の概要

コマンドライン引数とは、 プログラムを起動するときに、 外部から渡す値のことです。

コマンドライン引数 = プログラム起動時に渡されるデータ

Java では、コマンドライン引数は main メソッドの引数として受け取ります。

2. main メソッドとコマンドライン引数

Java プログラムの開始点である main メソッドは、 次の形で定義されています。 

public static void main(String[] args) {
}

この String[] args が、 コマンドライン引数を受け取るための配列です。

ポイント：
・引数はすべて String 型として渡される
・渡された順番どおりに配列に格納される

3. コマンドライン引数の渡し方

コマンドラインから Java プログラムを実行するとき、 プログラム名の後に値を並べて指定します。 

java Main 10 20 30

この場合、args 配列の中身は次のようになります。 

args[0] → "10"
args[1] → "20"
args[2] → "30"

数値を渡しても、
受け取った時点ではすべて String 型です。

4. サンプルプログラム

渡されたコマンドライン引数を順番に表示する例です。 

public class Main {

    public static void main(String[] args) {

        for (int i = 0; i < args.length; i++) {
            System.out.println(args[i]);
        }
    }
}

実行例： 

java Main Apple Banana Orange

実行結果： 

Apple
Banana
Orange

5. 数値として使う場合

コマンドライン引数は文字列として渡されるため、 数値として使いたい場合は変換が必要です。 

public class Main {

    public static void main(String[] args) {

        int a = Integer.parseInt(args[0]);
        int b = Integer.parseInt(args[1]);

        System.out.println(a + b);
    }
}

実行例： 

java Main 10 20

実行結果： 

30

注意：
数値に変換できない文字列が渡された場合、 実行時エラーが発生します。

6. よくある注意点

  • 引数が渡されない場合、args.length は 0 になる
  • args[0] を使う前に、長さの確認が必要
  • 配列なので、0 番目から始まる

まとめ

・コマンドライン引数は、プログラム起動時に外部から渡されるデータである。
・Java では main(String[] args) の args 配列で受け取る。
・引数はすべて文字列として渡され、順番どおりに配列に格納される。
・数値として使う場合は、文字列から数値への変換が必要である。
