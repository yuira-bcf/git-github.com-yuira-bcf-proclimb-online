A. 選択式問題（5問）

問1：コンストラクターの基本 基礎

次のうち、Javaのコンストラクターについて正しい説明はどれですか。

  1. コンストラクターは戻り値の型として void を書く。
  2. コンストラクター名はクラス名と一致する必要がある。
  3. コンストラクターは呼び出しが遅延され、最初にメソッドが呼ばれた時に実行される。
  4. コンストラクターは静的メソッドとして宣言しなければならない。
解答・解説

正解：B

コンストラクターはクラス名と同じ名前を持ち、戻り値は書きません。new による生成時に自動で呼び出されます。

問2：デフォルトコンストラクター 基礎

次のうち、デフォルトコンストラクターが自動生成される条件として正しいものはどれですか。

  1. フィールドが1つでも存在する場合、自動生成される。
  2. コンストラクターを1つも定義していない場合、自動生成される。
  3. final フィールドがある場合、自動生成される。
  4. クラスが public の場合のみ、自動生成される。
解答・解説

正解：B

コンストラクターを1つも書いていない場合に限り、引数なしコンストラクター（デフォルトコンストラクター）が自動生成されます。

問3：オーバーロード 基礎

次の説明のうち、コンストラクターのオーバーロードとして正しいものはどれですか。

  1. 同じ引数リストで、戻り値だけが異なるコンストラクターを複数定義する。
  2. 引数の数や型が異なる複数のコンストラクターを同じクラス内に定義する。
  3. 親クラスのコンストラクターを子クラスで上書きする。
  4. static を付けたコンストラクターを追加する。
解答・解説

正解：B

オーバーロードは、引数の数や型が異なる同名メソッド（コンストラクター）を複数用意することです。コンストラクターに戻り値はありません。

問4：this の役割 基礎

次のコードで this.name の this が表すものとして最も適切なものはどれですか。

class User {
    String name;

    User(String name) {
        this.name = name;
    }
}
  1. クラスそのもの（クラスオブジェクト）
  2. 現在生成中のインスタンス（このオブジェクト）
  3. 引数 name の別名
  4. 親クラスのインスタンス
解答・解説

正解：B

this は「現在のインスタンス」を表します。this.name はフィールドを指し、右側の name は引数です。

問5：引数なしコンストラクターの設計判断 応用

引数なしコンストラクターについて、設計として最も適切な考え方はどれですか。

  1. 常に定義するべきである。あるほど便利である。
  2. 原則として定義しない。必要な場合は理由と利用範囲を明確にして定義する。
  3. final フィールドがある場合でも必ず定義する。
  4. テストのためにのみ定義し、本番コードでは禁止する。
解答・解説

正解：B

引数なしコンストラクターは、必須項目が未設定のまま生成できる可能性があるため、原則として避けます。必要な場合は理由と利用範囲を設計で明確にします。

学習のポイント： 選択式では「文法の正しさ」だけでなく、「設計として適切か」を意識して判断できるようにしましょう。

B. プログラミング問題（5問）

問題1：必須項目をコンストラクターで初期化 基礎

次の条件を満たす Book クラスを作成してください。

  • フィールド：private final String title
  • コンストラクターで title を受け取り初期化する
  • getTitle() を用意してタイトルを返す
  • Main クラスで Book を生成し、タイトルを表示する
解答例 
class Book {
    private final String title;

    Book(String title) {
        this.title = title;
    }

    String getTitle() {
        return title;
    }
}

public class Main {
    public static void main(String[] args) {
        Book b = new Book("Java入門");
        System.out.println(b.getTitle());
    }
}

問題2：コンストラクターのオーバーロード 基礎

次の条件を満たす User クラスを作成してください。

  • フィールド：private final String name、private final int age
  • コンストラクターを2つ用意する
  • User(String name) は age を 0 として扱う
  • User(String name, int age) は両方を初期化する
  • Main で両方のコンストラクターを使って生成し、内容を表示する
解答例 
class User {
    private final String name;
    private final int age;

    User(String name) {
        this(name, 0);
    }

    User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    String info() {
        return name + " / " + age;
    }
}

public class Main {
    public static void main(String[] args) {
        User u1 = new User("Taro");
        User u2 = new User("Hanako", 20);

        System.out.println(u1.info());
        System.out.println(u2.info());
    }
}

問題3：入力値をチェックして初期化 応用

次の条件を満たす Product クラスを作成してください。

  • フィールド：private final String name、private final int price
  • コンストラクターで price が 0 未満の場合は 0 に補正する
  • info() を用意して「商品名: 価格」を返す
  • Main で price に -100 を渡して生成し、表示する
解答例 
class Product {
    private final String name;
    private final int price;

    Product(String name, int price) {
        this.name = name;
        this.price = (price < 0) ? 0 : price;
    }

    String info() {
        return name + ": " + price;
    }
}

public class Main {
    public static void main(String[] args) {
        Product p = new Product("Pen", -100);
        System.out.println(p.info()); // Pen: 0
    }
}

問題4：引数なしコンストラクターを避ける設計 応用

次の条件を満たす Member クラスを作成してください。

  • フィールド：private final String id（会員ID）
  • id は必須とし、引数なしコンストラクターは用意しない
  • Main で new Member() を書くとコンパイルエラーになることを確認できるようにする
  • 正しい生成例（new Member("A001")）では id を表示する
解答例 
class Member {
    private final String id;

    Member(String id) {
        this.id = id;
    }

    String getId() {
        return id;
    }
}

public class Main {
    public static void main(String[] args) {
        // Member m0 = new Member(); // ← 引数なしは用意していないのでコンパイルエラー

        Member m = new Member("A001");
        System.out.println(m.getId());
    }
}

確認：学習時は、コメントを外してコンパイルし、エラーになることを確認してから戻してください。

問題5：参照型フィールドをコンストラクターで受け取る 応用

次の条件を満たす Hero と Sword を作成してください。

  • Sword は private final String name を持ち、コンストラクターで初期化する
  • Hero は private final String name と private final Sword sword を持つ
  • Hero のコンストラクターで name と sword を受け取り初期化する
  • Main で剣を作り、勇者を作り、勇者の装備情報を表示する
解答例 
class Sword {
    private final String name;

    Sword(String name) {
        this.name = name;
    }

    String getName() {
        return name;
    }
}

class Hero {
    private final String name;
    private final Sword sword;

    Hero(String name, Sword sword) {
        this.name = name;
        this.sword = sword;
    }

    String info() {
        return name + " equips " + sword.getName();
    }
}

public class Main {
    public static void main(String[] args) {
        Sword s = new Sword("Bronze Sword");
        Hero h = new Hero("Taro", s);

        System.out.println(h.info());
    }
}
学習のポイント： コンストラクターは「生成時に正しい状態にする」ための仕組みです。必須項目・初期値・入力チェックを意識して実装してみましょう。
