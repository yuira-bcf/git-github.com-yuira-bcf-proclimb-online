# 06 成果物の保存と資産化

## 1. 何を保存すべきか

最低限、次を保存します。

- PROMPT_00〜PROMPT_09（実際に使った版）
- 生成された設計・要件（出力物）
- 実装コード（src/）
- README.md
- CHANGELOG.md

## 2. 「保存しておくと強い」追加物

- テスト結果（手動テストの実施ログ）
- 既知の制限（できていないこと）
- 次にやるべき改善メモ（Backlog）

## 3. CHANGELOG の書き方（最小形）

変更が発生したら、1件につきこれだけ書くと十分です。

- 変更内容（何を変えたか）
- 変更理由（なぜ変えたか）
- 影響範囲（どこが変わるか）
- 再テスト項目（どれを確認したか）

## 4. 資産化のゴール（Done）

「1か月後の自分」が、次を見て復元できる状態になっていれば成功です。

- 何を作ったか（README）
- どう動かすか（README）
- どの前提で作ったか（PROMPT_00〜）
- 何が変わったか（CHANGELOG）
