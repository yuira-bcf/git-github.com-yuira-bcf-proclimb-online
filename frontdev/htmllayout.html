1. レイアウトとは何か

CSS のレイアウトとは、ページ上の要素（文字・画像・見出し・ボタン・箱など）を、 「どこに」「どの大きさで」「どの順序で」「どのように並べるか」を決める仕組みです。

HTML の要素は、ブラウザの中では基本的に「四角い箱」として扱われます。
レイアウトの学習では、まず「箱の性質」を理解し、その上で「並べ方の道具（Flexbox、Grid など）」を使い分けることが重要です。
  • ボックスモデル：内容・内側余白・枠線・外側余白の考え方
  • 表示形式（display）：箱の並び方を決めるスイッチ
  • 通常フロー：何も指定しないときの並び方
  • Flexbox：横並びや縦並びなど、一次元の整列に強い方法
  • Grid：行と列の二次元配置に強い方法
  • 位置指定（position）：重ねる、固定する、貼り付けるための方法
  • レスポンシブ：画面幅や親要素の幅に応じてレイアウトを切り替えること
重要 レイアウトは「見た目」だけではなく、要素の配置ルール（並び方・サイズ計算・順序）を扱います。
学習の順序 ボックスモデル → 通常フロー → display → Flexbox / Grid → position → レスポンシブ
実務の主役 Flexbox と Grid が中心で、position は補助、float は回り込み目的で使います。

2. レイアウトの土台：ボックスモデル

2.1 ボックスモデルの構造

要素は、内側から外側へ次の順に広がります。

  1. 内容（content）：文字や画像などの中身
  2. 内側余白（padding）：内容の周囲の余白
  3. 枠線（border）：境界線
  4. 外側余白（margin）：他の要素との距離

2.2 幅の計算を安定させる設定

レイアウトでつまずきやすい理由の一つは、「指定した幅と見た目の幅がずれる」ことです。 そのため、次の設定を基本として使用することが多いです。 

* {
  box-sizing: border-box;
}

border-box を指定すると、width に内側余白（padding）と枠線（border）が含まれるため、サイズ計算が安定します。

3. レイアウトの出発点：通常フロー

CSS を特に指定しない場合、要素は「通常フロー（通常の並び方）」で配置されます。

ブロック要素
上から下へ縦に並びます。幅は基本的に親要素いっぱいになります。
例：div、p、h1
インライン要素
文章のように左から右へ横に流れます。幅や高さの指定は効きにくいです。
例：span、a

通常フローを理解しておくと、Flexbox や Grid を使ったときに「何が変わったのか」を説明しやすくなります。

4. display（表示形式）はレイアウトのスイッチです

display は、要素の並び方を大きく変える重要な指定です。

  • display: block;：縦に積まれる（ブロック要素として振る舞います）
  • display: inline;：横に流れる（インライン要素として振る舞います）
  • display: inline-block;：横に並びますが、幅・高さを持てます（扱いに癖があります）
  • display: flex;：Flexbox（一次元の並びに強い方法）
  • display: grid;：Grid（行と列の二次元配置に強い方法）
実務の中心：flex / grid 基礎理解：block / inline 補助知識：inline-block

5. Flexbox（フレックスボックス）：一次元の並びに強い方法

5.1 Flexbox の考え方

Flexbox は、親要素（フレックスコンテナ）の中で、子要素（フレックスアイテム）を並べる仕組みです。
「一次元」とは、横方向（行）または縦方向（列）のどちらか一方向を中心に整列する、という意味です。

5.2 よく使う指定

  • display: flex;：Flexbox を開始します（親要素に指定します）
  • flex-direction：並ぶ向きを決めます（横または縦）
  • justify-content：並ぶ向き（主方向）での揃え方を決めます
  • align-items：主方向と直角の向き（交差方向）での揃え方を決めます
  • gap：要素間のすき間を指定します
  • flex-wrap：入りきらないときに折り返すかどうかを決めます

5.3 例：横並びメニュー（中央寄せ）

.menu {
  display: flex;
  justify-content: center; /* 左右方向の中央揃え */
  align-items: center;     /* 上下方向の中央揃え */
  gap: 16px;               /* リンク同士の間隔 */
  height: 56px;
}
Flexbox は、上下左右の中央揃えや、均等な間隔の並びを作るのが得意です。
「中央寄せが難しい」という昔の悩みは、Flexbox でかなり解決できます。

6. Grid（グリッド）：行と列の二次元配置に強い方法

6.1 Grid の考え方

Grid は、親要素に「行」と「列」を定義し、そのマス目の中へ要素を配置する仕組みです。
ページ全体の骨組み（ヘッダー・本文・サイドバー・フッター）を作るのに向いています。

6.2 よく使う指定

  • display: grid;
  • grid-template-columns：列の幅を決めます
  • grid-template-rows：行の高さを決めます
  • gap：マス目のすき間を決めます
  • grid-template-areas：名前で配置を指定できます（教科書向きです）
  • fr：残りの空間を分配する単位です

6.3 例：2列（左 240px、右は残り全部）

.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  gap: 16px;
}
Grid は「行と列の両方を設計する」ため、ページ全体の構造を作るのが得意です。

7. position（位置指定）：固定・重ね・貼り付けに使う方法

position は、要素を通常フローから外したり、指定した位置へ固定したりするために使います。
ただし、ページ全体の骨組みは Grid や Flexbox の方が分かりやすいため、位置指定は「特殊効果」として使うのが基本です。

  • position: relative;：現在の位置を基準にします（absolute の基準作りにも使用します）
  • position: absolute;：基準（多くは relative を指定した親）からの位置で配置します
  • position: fixed;：画面（表示領域）に固定します
  • position: sticky;：スクロール中に特定位置へ吸着します（条件があります）
注意： position を多用してページ全体を組み立てると、後から修正しづらくなることがあります。
レイアウトの骨組みは Grid / Flexbox、重ねや固定は position、と役割分担すると整理しやすいです。

8. float（フロート）：文章の回り込みに使う方法

float は、現在ではページ全体のレイアウトには通常使いません。
本来の目的は、文章の途中にある画像に文章を回り込ませることです。

  • 回り込み：float: left; など
  • 回り込み解除：clear: both; など
float は「文章と画像の回り込み」を学ぶときに、役割がはっきりして理解しやすい方法です。

9. レスポンシブ（画面幅や親要素の幅への対応）

9.1 メディアクエリ（画面幅で切り替える）

メディアクエリは、画面の幅が条件を満たすときに CSS を切り替える仕組みです。 

@media (max-width: 768px) {
  /* ここにスマートフォン向けの指定を書きます */
}

9.2 コンテナクエリ（親要素の幅で切り替える）

コンテナクエリは、画面全体ではなく、部品（親要素）の幅を基準にして CSS を切り替える仕組みです。
同じ部品を別の場所に置いたときでも、置かれた場所の幅に応じて見た目を変えられます。 

.card-area {
  container-type: inline-size;
}

@container (max-width: 500px) {
  /* 親要素の幅が 500px 以下の場合の指定 */
}
教材では「画面幅での切り替え（メディアクエリ）」から学び、慣れてから「親要素の幅での切り替え（コンテナクエリ）」へ進むと理解がスムーズです。

10. 使い分けの目安（まとめ）

  • 横並び・縦並び・中央揃え・均等配置・折り返し：Flexbox
  • ページ全体の区画・行と列の設計・二次元の配置：Grid
  • 重ねる・固定する・右上にバッジを置く：position
  • 文章に画像を回り込ませる：float
  • 画面幅で切り替える：メディアクエリ
  • 親要素の幅で切り替える：コンテナクエリ
迷ったら：まず Flexbox または Grid を検討します position は補助として使用します float は回り込み目的で使用します
